## 2025-07-04 - [DOM Caching & Line Counting Efficiency]
**Learning:** In web-based editors, frequent DOM lookups (e.g., `document.getElementById`) in event handlers like `onkeyup` or `oninput` cause measurable performance degradation. Caching these elements during initialization can improve responsiveness by ~25%. Additionally, while `split('\n')` is often seen as inefficient due to array allocation, it is surprisingly fast in modern JavaScript engines compared to manual loops or regex matching for simple line counting tasks on medium-sized strings (<100k chars).
**Action:** Always cache frequently accessed DOM elements as class properties in the `constructor` or `setup` phase. Before replacing "inefficient" string methods with manual logic, benchmark to ensure the alternative is actually faster in the target environment.
